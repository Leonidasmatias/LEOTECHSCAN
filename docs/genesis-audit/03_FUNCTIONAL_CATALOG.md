## Observações gerais

- `operator-engine/` (diretório) contém apenas um `README.md` de 126 bytes — o nome sugere um motor de operadora dedicado, mas não existe código ali; a lógica de operadora real está em `lib/operator.ts`. Classificação: **NÃO IMPLEMENTADO** (como módulo separado — a capacidade em si existe, mas não neste diretório).
- `database/schema.ts` (5 linhas) não é um schema de banco real — é um arquivo de constantes que reexporta nomes de `core/site.ts`. O schema real (DDL) vive em `importers/multi_operator_import.py` (`CREATE_SQL`) e em `services/data-trust-engine.ts` (`ensureDataTrustTables`, que cria tabelas sob demanda com `CREATE TABLE IF NOT EXISTS`). Isso significa que **o schema do banco está fragmentado em pelo menos três lugares diferentes do código-fonte**, sem uma única fonte de verdade nem sistema de migração formal.
