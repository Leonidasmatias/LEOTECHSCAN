## Verificações pedidas explicitamente pela missão

- **O frontend recalcula dados?** NÃO FOI POSSÍVEL CONFIRMAR em profundidade — os componentes React não foram lidos linha a linha nesta auditoria (e vários estão com JSX quebrado no working tree atual, ver `11_TESTS_AND_QUALITY.md`), então não é possível afirmar com evidência direta se algum componente refaz algum cálculo de score no cliente. Nenhuma rota de API observada delega cálculo de score para o cliente (todos os scores lidos são computados server-side).
- **Diferentes rotas usam fórmulas diferentes para o "mesmo" conceito?** SIM, um caso confirmado: `risco`/`ori_risk` (persistidos no import) coexistem com o `risk` calculado em runtime no dashboard (`app/api/dashboard/route.ts`, baseado em faixas de `geo_score`: ≤30 baixo, 31–60 médio, 61–80 alto, ≥81 crítico) — são dois conceitos de "risco" com nomes parecidos, potencialmente de origens diferentes. Não foi possível confirmar nesta auditoria se produzem o mesmo veredito para o mesmo site.
- **Scores com mesmo nome, fórmulas diferentes?** Não confirmado nenhum caso direto (LTS/TCI/OPI/SRI têm nome único e fórmula única localizada); "Risk"/"Risco" (ver acima) é o caso mais próximo de uma sobreposição de nomenclatura.
- **Pesos hardcoded no código (não externalizados)?** SIM — `calculateTci` (pesos 35/25/20/20) e `confidenceForSite` (pesos 0.20/0.12/0.12/0.10/0.10/0.16/0.10/0.10) têm pesos diretamente no código-fonte, enquanto LTS/OPI/SRI usam `config/sentinel_rules.json`. Inconsistência confirmada por leitura direta dos dois arquivos.
- **Thresholds espalhados?** SIM — limites como "crítico se `geo_score >= 81`" aparecem tanto em `config/sentinel_rules.json` (`limits.alertCriticalSri`, `alerts.criticalSri` = 81) quanto hardcoded em `app/api/dashboard/route.ts` (`geo_score >= 81`) e em `services/confidence-engine.ts` (`site.geoScore || 0) >= 81 ? 8`). O valor 81 é consistente entre eles (bom sinal), mas está duplicado em pelo menos três lugares em vez de referenciar uma única constante.
- **Valores arbitrários sem justificativa?** SIM, confirmados: fallback `lowDensity = 40` em `calculateOpi` quando população é zero; degraus fixos de redundância (1/3/8 sites) em `calculateSri`; peso `+10` fixo somado incondicionalmente na fórmula de Trust Score (`... + 10 - duplicate - alertPenalty`), sem comentário explicando a origem desse valor.
- **Vazamento temporal (usar dado do futuro)?** Não identificado nos cálculos lidos — todos operam sobre o estado atual do banco no momento da chamada, sem parâmetro de data de corte, o que também significa que não há como recalcular um score "como ele era" em uma data passada (ausência de versionamento temporal, não vazamento).
- **Inferências apresentadas como fatos?** SIM, um caso relevante e auto-reconhecido no próprio código: `services/data-trust-engine.ts::dataTrustDashboard` retorna `governance: "Trust Score e uma estimativa tecnica; nao substitui vistoria de campo..."` — o próprio serviço já se declara como estimativa, o que é uma boa prática. Já a "Telecom AI" (ver `09_AI_AND_MACHINE_LEARNING.md`) responde perguntas em linguagem natural formatada como uma resposta de IA quando na verdade é correspondência de palavras-chave contra um conjunto fixo de intents — isso tem risco de ser lido pelo usuário final como uma inferência mais sofisticada do que realmente é.
- **Classificações sem confiança associada?** `risco`/`ori_risk` (persistidos no import) e o `risk` do dashboard não carregam um valor de confiança associado — diferente de `Trust Score` e `Coordinate Quality`, que explicitamente carregam confiança (`overallConfidence`, `confidence`).

Status geral desta seção: **CONFIRMADO NO CÓDIGO** para todas as fórmulas de `services/sentinel-scoring.ts`, `confidence-engine.ts`, `data-trust-engine.ts` e `coordinate-quality-engine.ts` (lidos integralmente). **NÃO FOI POSSÍVEL CONFIRMAR** a fórmula exata de `geo_score` e `ori_score` no importador Python (lido apenas parcialmente) e a fórmula de `satellite-validation-engine.ts` (não lido nesta passagem) — ambos ficam como pendência explícita para a próxima rodada de leitura.
