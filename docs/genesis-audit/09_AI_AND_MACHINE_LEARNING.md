# 09 — IA e Machine Learning

## Classificação geral

**Nenhuma capacidade de Machine Learning real (treinamento de modelo, embeddings, classificação estatística aprendida, regressão) foi encontrada em nenhum arquivo lido nesta auditoria.** Todas as capacidades rotuladas como "IA", "Inteligência" ou "Recomendação" no projeto são heurísticas determinísticas ou regras de negócio explícitas, escritas diretamente em TypeScript, sem dataset de treino, sem features aprendidas, sem processo de validação estatística, e sem dependência de nenhuma biblioteca de ML (`package.json` não lista `tensorflow`, `scikit-learn`/equivalente JS, `onnxruntime`, nem SDK de LLM/API de IA externa).

## Para cada capacidade "AI-like" encontrada — objetivo, dataset, features, treino, métricas

Como nenhuma das capacidades acima constitui ML real, os campos pedidos pela missão (dataset, features, target, treinamento, validação, métricas, risco de vazamento, persistência de modelo, versionamento de modelo) **não se aplicam** — não há modelo a persistir ou versionar. Isso é reportado explicitamente como resposta, não como omissão: a pergunta "existe ML real aqui?" tem resposta **NÃO** para todos os módulos revisados nesta auditoria.

## Fallback e explicabilidade

Todas as heurísticas revisadas são inerentemente explicáveis (são `if`/soma ponderada, não uma caixa-preta) — o que é uma vantagem real para governança, mesmo não sendo "IA" no sentido técnico do termo. O risco não é de opacidade algorítmica; é de **nomenclatura**: chamar de "Telecom AI" ou "Sentinel Intelligence Core" um conjunto de regras condicionais pode levar um usuário executivo a atribuir a essas respostas uma confiança maior do que a heurística realmente sustenta. Isso é relevante para o Sentinel-1 Enterprise: se a plataforma pretende oferecer IA real (ex.: um modelo de previsão de expansão de rede, ou um LLM para responder perguntas em linguagem natural livre), isso precisa ser construído do zero — nada no código atual pode ser "ligado" para virar ML real; o que existe é uma boa base de features (os scores e agregações já calculados) que poderia alimentar um modelo real no futuro.

Status desta seção: **CONFIRMADO NO CÓDIGO** para os módulos lidos integralmente (Telecom AI, Confidence, Data Trust, scores municipais, Coordinate Quality); **PARCIALMENTE CONFIRMADO** (por tamanho de arquivo e padrão, não leitura linha a linha completa) para Inference Engine, Recommendation Engine e Copernicus.
